GREEN COMMITS AUTOMATION SYSTEM - UPDATED GUIDE
===========================================

This guide explains how to use the Green Commits Automation System to maintain an active GitHub contribution graph.

OVERVIEW
--------
The system automatically makes commits to your GitHub repository on scheduled days with varying intensity levels to create a natural-looking contribution graph. It includes features for:

1. Simple commits (morning session)
2. Full GitHub workflow with PRs, issues, and code reviews (afternoon session)
3. Backdating commits to fill in your contribution history
4. Automatic scheduling with natural patterns and dark green squares twice per week

SETUP INSTRUCTIONS
-----------------
1. Clone this repository to your local machine
2. Run the fix_automation.ps1 script as administrator to set up the scheduled tasks:
   - Right-click on fix_automation.ps1
   - Select "Run with PowerShell" or "Run as administrator"

3. The script will create four scheduled tasks:
   - Green GitHub Workflow - Morning (runs daily at 9:00 AM)
   - Green GitHub Workflow - Afternoon (runs daily at 3:00 PM)
   - Green GitHub Workflow - Startup (runs when your computer starts)
   - Green GitHub Workflow - Logon (runs when you log in)

MANUAL OPERATION
---------------
If you need to run the workflow manually:

1. Double-click on run_green_commits.bat
2. Choose "morning" or "afternoon" session when prompted
3. The script will run the appropriate workflow

BACKDATING COMMITS
-----------------
To fill in your contribution graph for the past year:

1. Run backdate_commits.ps1:
   - Right-click on backdate_commits.ps1
   - Select "Run with PowerShell"

2. The script will:
   - Generate a natural-looking commit schedule for the past year
   - Create backdated commits with varying intensity levels
   - Push the commits to GitHub (with your confirmation)

TROUBLESHOOTING
--------------
If the automation stops working:

1. Run fix_automation.ps1 again to reset the scheduled tasks
2. Check Task Scheduler to ensure the tasks are enabled
3. Verify that the last_run.txt file is not preventing execution
4. Run the workflow manually using run_green_commits.bat to test

CONFIGURATION
------------
The system uses several configuration files:

- commit_schedule.txt: Contains the dates scheduled for commits
- commit_intensity.txt: Maps dates to intensity levels (2-5)
- last_run.txt: Records when the script last ran

INTENSITY LEVELS
--------------
The system uses 4 intensity levels for GitHub contributions:

- Level 2: Medium-light green (1-2 commits)
- Level 3: Medium green (3 commits)
- Level 4: Medium-dark green (4 commits)
- Level 5: Dark green (5 commits)

Dark green squares (levels 4-5) are scheduled to appear twice per week for a natural pattern.

IMPORTANT NOTES
--------------
1. The system will only run when your computer is on
2. If you miss scheduled commits, they will not be made retroactively
3. The backdating script should be used with caution as it modifies Git history
4. Always keep a backup of your repository before making significant changes

For any issues or questions, please refer to the script comments or contact the repository owner.
